# e2e-trip-crud-tests

## Goal
Create E2E tests using Playwright that verify the CRUD functionality of trips for a vehicle. The existing auth.setup.ts should serve as a reliable setup hook for all future tests, starting with trip CRUD tests.

- Related spec `021-e2e-trip-crud-panel-tests` exists and can inform requirements

### Key Findings

**Best Practices:**
- Home Assistant requires UI-based Config Flow installation (not just login)
- Shadow DOM requires web-first locators: getByRole, getByText, getByLabel
- auth.setup.ts pattern: login + Config Flow + storageState save
- Anti-patterns: waitForTimeout, CSS/XPath selectors, hardcoded URLs

**CRUD Operations to Test:**
- Create: trip_create service via "+ Agregar Viaje" button
- Read: trip_list service displaying trips
- Update: trip_update via "Editar" button
- Delete: delete_trip via "Eliminar" + confirmation
- Pause/Resume: pause_recurring_trip/resume_recurring_trip services
- Complete/Cancel: complete_punctual_trip/cancel_punctual_trip services

- 021-e2e-trip-crud-panel-tests: Same goal, broader scope (use its requirements as reference)

## Learnings
- Requirements adapted from 021 spec but focused on trip CRUD only (skipped panel loading tests US-1, US-2 from 021)
- Trip CRUD user stories: US-1 List, US-2 Create, US-3 Edit, US-4 Delete, US-5 Pause/Resume, US-6 Complete/Cancel
- auth.setup.ts must not be modified - provides storageState only, tests reuse it
- Panel URL pattern: `/ev-trip-planner-{vehicle_name}` (e.g., Coche2)
- Service names: trip_create, trip_update, delete_trip, pause_recurring_trip, resume_recurring_trip, complete_punctual_trip, cancel_punctual_trip
- UI labels: "+ Agregar Viaje", "Editar", "Eliminar", "Pausar", "Reanudar", "Completar", "Cancelar"
- Shadow DOM requires web-first locators (getByRole, getByText, getByLabel) - no CSS/XPath
