## Learnings from Design

- BUG-1 and BUG-2 are the same bug: duplicate panel registration causes flickering
- config_flow.py:839-851 registers panel in _async_create_entry (called ONCE during setup)
- __init__.py:652 registers panel in async_setup_entry (called on EVERY reload)
- Solution: Remove panel registration from config_flow.py, keep only in __init__.py with normalized vehicle_id
- BUG-5 is in EMHASSAdapter, not TripManager - adapter uses vehicle_name for sensor naming instead of entry_id
- EMHASSAdapter.__init__ receives Dict[str,Any] (entry.data), not ConfigEntry - needs refactoring to receive ConfigEntry
- Dashboard templates hardcode `sensor.{{ vehicle_id }}_soc` - but user configures custom soc_sensor in config flow
- kWh auto-calculation requires dashboard template changes + input helpers to be readonly (mode: display)
