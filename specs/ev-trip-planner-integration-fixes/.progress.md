# Progress: ev-trip-planner-integration-fixes

## Original Goal

Correcciones y mejoras en la integración EV Trip Planner:
1. **Panel duplicado**: Al agregar un coche via config flow, se crean dos paneles con URLs que difieren solo en mayúsculas (ev-trip-planner-chispitas vs ev-trip-planner-Chispitas). Crear test que detecte el error, luego la solución y validar.
2. **Panel parpadeando**: El panel del vehículo se actualiza constantemente en un bucle agresivo, llenando la consola de mensajes debug repetitivos.
3. **Campo energía estimada**: Debe ser solo lectura y calcularse automáticamente según distancia × consumo del vehículo.
4. **Cascada de eliminación**: Al borrar una integración deben eliminarse todos sus viajes asociados.
5. **Índice p_deferrable por viaje**: Cada viaje debe mostrar su índice (0, 1, 2...) en la tarjeta, ya que EMHASS devuelve planificación por separado para cada p_deferrable.

**EXTENDED SCOPE (added 2026-03-29):**
6. **Display p_deferrable schedule**: Mostrar gráfico de planificación de carga en tarjeta del viaje
7. **EMHASS JSON format**: Transformar sensor deferrables_schedule al formato que EMHASS entiende
8. **Automation template**: Crear automatización para controlar carga según plan EMHASS
9. **Charging window calculation**: Ventana de carga = fin viaje anterior hasta inicio viaje siguiente (6hr gap)
10. **SOC integration**: Mejorar sensor emhass_perfil_diferible con SOC del coche
11. **Pre-charging logic**: Si ventana insuficiente, pre-cargar en ventana anterior
12. **SOC indicators**: Mostrar SOC esperado al inicio y fin de cada viaje

## Learnings

- Issue 1 (Duplicate Panels): Panel registered twice - once in config_flow._async_create_entry() and again in __init__.async_setup_entry(). Both use same vehicle_id (lowercased), but HA's panel registration may treat URLs as case-sensitive, creating two panels.
- Issue 2 (Panel Flickering): DataUpdateCoordinator with update_interval=30s but services call async_request_refresh() on every trip operation. No debouncing/throttling implemented.
- Issue 3 (Estimated Energy): kwh field is user-provided but should be calculated from km x consumption. calcular_energia_kwh() exists in utils.py but not used as primary source.
- Issue 4 (Cascade Delete): async_unload_entry() in __init__.py does NOT delete trips from TripManager storage. Trips remain orphaned in HA storage after integration removal.
- Issue 5 (p_deferrable Index): EMHASS adapter assigns indices to trips (async_assign_index_to_trip) and stores in _index_map, but index is NOT persisted with trip data. Index assigned at publish time, not stored with trip.
- panel_custom.py line 16-24 registers a static panel at "ev-trip-planner" URL - this is separate from per-vehicle dynamic panels
- vehicle_id generation: vehicle_name.lower().replace(" ", "_") in config_flow.py line 725
- TripManager uses HA Store API for persistence (not raw file I/O)
