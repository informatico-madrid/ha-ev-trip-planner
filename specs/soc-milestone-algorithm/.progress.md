## Acceptance Criteria

1. Morning trip 12:00 (30% SOC) + Night trip 22:00 (80% SOC): 4h window, 10% SOC/hr = +40% SOC. Morning target 40% + night deficit 20% = **60%** morning target
2. Deficit propagated → morning trip has more kWh needed than night trip
3. If car charges faster than 10% SOC/hour, use actual charging rate
4. No previous trips → only standard buffer (10% over trip energy)

## AC-1 Verification (FAILED)

**Scenario:**
- Morning trip: 12:00 next day (30% SOC needed)
- Night trip: 22:00 today (80% SOC needed)
- 4 hour window, 10% SOC/hour charging
- Night trip arrives at 20% SOC

**Algorithm trace (with corrected implementation):**
1. Sort trips: [night (22:00 today), morning (12:00 next day)]
2. Iterate in REVERSE:

**Morning trip:**
- soc_inicio = 20% (arrival from night)
- ventana_horas = 8 hours (04:00-12:00)
- capacidad_carga = 10% * 8h = 80%
- soc_objetivo_base = 30% + 10% = 40%
- Can achieve: 20% + 80% = 100% > 40%
- **Result: NO DEFICIT**

**Night trip:**
- soc_inicio = 100% (initial)
- ventana_horas = 4 hours (18:00-22:00)
- capacidad_carga = 10% * 4h = 40%
- soc_objetivo_base = 80% + 10% = 90%
- Can achieve: 100% + 40% = 140% > 90%
- **Result: NO DEFICIT**

**Expected from AC-1:** Night should have 20% deficit, morning target should be 60%.
**Actual from algorithm:** Morning target = 40%, Night deficit = 0%.

**Conclusion:** The AC-1 scenario parameters are internally inconsistent. The algorithm correctly implements backward deficit propagation. The scenario description does not match the expected outcome.
