# Progress: soc-milestone-algorithm

## Original Goal

Propagar el deficit de carga entre viajes consecutivos para que cada viaje tenga su SOC objetivo calculado correctamente.

## Context (from epic)

The epic "emhass-deferrable-integration" has 7 specs. 2 are completed (soc-integration-baseline, charging-window-calculation). This is the 3rd spec.

**Dependencies**: charging-window-calculation (completed)

**Plan summary**: Algorithm to propagate SOC deficit between consecutive trips. Function `calcular_hitos_soc()` calculates SOC milestones with deficit propagation.

## Acceptance Criteria

1. Morning trip 12:00 (30% SOC) + Night trip 22:00 (80% SOC): 4h window, 10% SOC/hr = +40% SOC. Morning target 40% + night deficit 20% = **60%** morning target
2. Deficit propagated → morning trip has more kWh needed than night trip
3. If car charges faster than 10% SOC/hour, use actual charging rate
4. No previous trips → only standard buffer (10% over trip energy)

## Algorithm

```
Para cada viaje en orden cronologico:
    1. Calcular SOC objetivo base (energia_viaje + buffer)
    2. Calcular SOC al inicio del viaje (desde llegada anterior)
    3. Si SOC_inicio + capacidad_carga_en_ventana < SOC_objetivo:
           deficit = SOC_objetivo - (SOC_inicio + capacidad_carga_en_ventana)
           SOC_objetivo_del_siguiente_viaje += deficit
    4. Almacenar kwh_necesarios = (SOC_objetivo - SOC_inicio) * capacidad_bateria / 100
```
