## Learnings
- Lit web component uses Shadow DOM - selectors must use `>>` pierce syntax
- Panel URL pattern: `/ev-trip-planner-{vehicle_id}` (extracts vehicle_id from URL)
- HA services called via `hass.callService('ev_trip_planner', service_name, data)`
- Services: `trip_create`, `trip_update`, `delete_trip`, `trip_list`, `trip_get`
- Form types: recurrente (day+time) and puntual (datetime-local)
- Delete confirmation uses native `window.confirm()` - Playwright dialog handler needed
- GitHub Actions workflow already exists at `.github/workflows/playwright.yml`
- Tests should run in `tests/e2e/` directory per workflow config
- Ephemeral HA setup via globalSetup writes `server-info.json` for tests to read
- **Config Flow required in auth.setup.ts** - integration must be installed via UI or panel gives 404
- **Dialog handling**: `page.on('dialog', async dialog => { await dialog.accept(); })` for `window.confirm()`
- **CSS pierce syntax**: `page.locator('ev-trip-planner-panel >> .selector')` traverses Shadow DOM
- **Vehicle ID hardcoded** as "test_vehicle" for CI consistency
