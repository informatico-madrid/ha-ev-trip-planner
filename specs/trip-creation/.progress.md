# Trip Creation

## Goal

Implement trip creation functionality for the EV Trip Planner panel. The panel currently has a "Add Trip" button that opens a modal, but saving doesn't work. Need to implement proper save, edit, and delete functionality with full test coverage.

## Research Phase

**Phase Status:** ✅ Complete

**Research Output:** `./specs/trip-creation/research.md`

**Key Findings:**

1. **Form Handling Patterns:**
   - Use `e.preventDefault()` to prevent page refresh
   - Use `FormData` API for reliable data extraction
   - Validate required fields before service call
   - Validate numeric fields for positive values

2. **Service Call Patterns:**
   - Standard pattern: `await this._hass.callService(domain, service, serviceData)`
   - HA service responses come directly, not wrapped in `{result: {...}}`
   - All CRUD services already implemented in backend

3. **Error Handling:**
   - Use try/catch/finally for robust error handling
   - Show loading state on submit button during operations
   - Provide user feedback via alerts
   - Fallback to console logging for debugging

4. **Panel Implementation:**
   - Lit web components with reactive properties
   - Polling for hass initialization
   - State subscription for real-time updates
   - Multiple fallback methods for URL parameter extraction

5. **Available Services:**
   - `ev_trip_planner.trip_list` - List all trips
   - `ev_trip_planner.trip_create` - Create new trip
   - `ev_trip_planner.trip_update` - Update existing trip
   - `ev_trip_planner.delete_trip` - Delete trip
   - Additional services for pause/resume/complete/cancel

**Feasibility:** High | **Risk:** Low | **Effort:** Medium (M)

## Design Phase

**Phase Status:** ✅ Complete

**Design Output:** `./specs/trip-creation/design.md`

**Approach:** Extend existing service layer with enhanced frontend handling using Lit web component patterns

**Components:**
- **EV Trip Planner Panel**: Main Lit component hosting form modal and trip list
- **Trip Form Modal**: Form interface with type selector (recurring/punctual)
- **Form Validation Layer**: Real-time validation of required fields
- **Service Call Handler**: Encapsulates callService logic with error handling
- **Trip List Display**: Renders trip cards with edit/delete buttons

**Key Decisions:**
- Use `e.preventDefault()` with form event listeners for consistent handling
- Wrap all service calls in try-catch-finally for reliable error handling
- Call `_loadTrips()` after every CRUD operation for UI sync
- Pre-fill form in edit mode using same modal for create/edit
- Use standard dialog pattern for delete confirmation
- Show loading state on submit button during service calls
