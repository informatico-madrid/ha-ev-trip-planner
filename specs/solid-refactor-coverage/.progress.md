### Goal Interview
- **Problem being solved**: trip_manager.py (~1346L) y emhass_adapter.py (~1536L) son God Classes con lógica de negocio mezclada con I/O. 100% coverage es imposible sin doubles profundamente anidados.
- **Constraints and must-haves**: TripManager y EMHASSAdapter deben mantener backward compatibility en interfaz pública. Storage format no puede cambiar. Tests existentes deben pasar.
- **Success criteria**: coverage 100% como consecuencia del buen diseño; tests usan doubles correctos según Layered Test Doubles Strategy
- **Scope boundaries**: IN — pure functions extraction, Protocol DI, Layer 1 migration; OUT — split en subclases nuevas (no es el objetivo)
- **User's existing knowledge**: Ya sabe que MagicMock() sin spec es violación, que Fakes van en tests/__init__.py, que Layer 3 en conftest.py es solo fixtures

### Approach Selection
- **Chosen approach**: A → B → C → D incremental con Protocol DI
  - Fase A: Pure functions → calculations.py / utils.py
  - Fase B: TripStorageProtocol + EMHASSPublisherProtocol → protocols.py
  - Fase C: Inject via constructor, default real impl
  - Fase D: Migrate Layer 1 + fix MagicMock() violations incrementally
- **Rationale**: Funciones puras primero (fácil 100% cover, sin doubles), luego Protocols (DI sin reescribir clases), luego cleanup tests

## Learnings

- **Phase ordering matters**: Extracting pure functions (Phase A) first is the right approach — it provides immediate test coverage wins without introducing DI complexity
- **Protocol DI is non-invasive**: Using `typing.Protocol` allows adding DI without modifying existing classes — TripManager and EMHASSAdapter don't need to inherit from anything
- **MagicMock() without spec is pervasive**: Over 250 instances found across test files — the violations are concentrated in test_emhass_adapter.py, test_dashboard.py, and test_trip_manager.py
- **tests/__init__.py is empty but documented**: The TDD_METHODOLOGY.md already contains the exact pattern that should be in tests/__init__.py — the content just needs to be created
- **calculations.py already has some pure functions**: `calculate_day_index`, `calculate_trip_time`, `calculate_charging_rate`, `calculate_soc_target` etc. are already extracted — Phase A should verify which TripManager/emhass_adapter functions are truly new extractions vs already done
- **Backward compatibility constraint is critical**: TripManager already has `set_emhass_adapter()` / `get_emhass_adapter()` — constructor injection must preserve this pattern
- **HA integration test complexity**: This is a `fullstack` project (HA integration with EMHASS HTTP API calls) — test doubles strategy must account for both the HA event bus and external HTTP dependencies
- **Testing Discovery Checklist passed**: pytest runner confirmed working via smoke test on test_trip_manager_core.py (47 tests pass). Coverage command: `pytest --cov=custom_components.ev_trip_planner`. ruff/mypy configured in pyproject.toml
- **utils.py already has trip_id generation functions**: `generate_trip_id()`, `is_valid_trip_id()`, `calcular_energia_kwh()` already exist in utils.py — Phase A for utils.py focuses on time validation/sanitization functions
- **calculations.py already has core SOC/window functions**: `calculate_day_index`, `calculate_trip_time`, `calculate_charging_rate`, `calculate_soc_target`, `calculate_energy_needed`, `calculate_charging_window_pure`, `calculate_multi_trip_charging_windows`, `calculate_soc_at_trip_starts`, `calculate_deficit_propagation`, `calculate_power_profile` all exist — EMHASSAdapter pure functions need to be added
- **AC-C1.1 constraint is strict**: Constructor takes `storage: TripStorageProtocol` NOT `| None = None` — default created inline as `storage if storage is not None else YamlTripStorage(...)` without if-branch in constructor body
