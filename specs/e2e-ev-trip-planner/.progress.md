## Learnings
- **hass-taste-test limitation**: The library is designed for Lovelace card testing. The EV Trip Planner is a **native HA panel** (via `panel_custom.async_register_panel`), not a Lovelace card. You cannot use `hass.Dashboard()` or `HassCard` - must navigate directly to panel URL and use raw Playwright page object.
- **Panel URL pattern**: `/ev-trip-planner-{vehicle_id}` where vehicle_id is an HA-generated UUID, not the vehicle name.
- **Shadow DOM pierce**: Use `>>` combinator: `page.locator('ev-trip-planner-panel >> .selector')`. Built-in locators pierce shadow DOM automatically in light DOM contexts.
- **Config Flow required**: EV Trip Planner must be installed via UI Config Flow (not Lovelace resource). This is the E2E test's job - global.setup.ts does NOT install the integration.
- **No playwright.config.ts**: Project has no Playwright config file. Tests reference `tests/e2e/` directory which does not exist yet.
- **Critical unknown**: After Config Flow, how to extract the vehicle_id to navigate to the panel URL. Sidebar navigation (Option C) is recommended over direct URL.
- **hass-taste-test v0.2.7** source is in `node_modules/` - fully examined. Creates ephemeral HA in temp dirs, uses lockfile for parallel coordination, auto-onboards dev/dev user.
- **Global setup already exists**: `tests/global.setup.ts` starts ephemeral HA, copies panel.js to www/, saves server-info.json. `tests/global.teardown.ts` cleans up.
- **getDiffable-html.js** traverses Shadow DOM automatically when generating HTML snapshots - used by HassCard.html() for visual regression.
