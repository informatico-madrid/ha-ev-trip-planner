### Problemas a Resolver

1. **Dos dispositivos creados**: El sensor `EmhassDeferrableLoadSensor` usa `entry_id` en lugar de `vehicle_id` en su `device_info`, creando dos dispositivos separados en Home Assistant:
   - "EV Trip Planner chispitas" (7 entidades) - correcto
   - "EV Trip Planner 01KNS8EEZCHZD8HZM4WGJEKPJJ" (1 entidad) - incorrecto

2. **Atributos vacíos**: El sensor tiene `null` en todos sus atributos:
   - `power_profile_watts`: null (debería ser array de 168 valores)
   - `deferrables_schedule`: null (debería ser lista con timestamps)
   - `emhass_status`: null (debería ser "ready", "active", etc.)

3. **Actualización**: El sensor debe actualizarse cuando:
   - Cambia el SOC del vehículo
   - Se crean/modifican/borran viajes
   - La hora actual avanza (rotación del perfil)

4. **Rotación horaria**: El primer valor del `power_profile_watts` debe representar siempre la **siguiente hora** completa (ej: si son las 9:30, el primer valor es de 10:00-11:00).

### Especificación Esperada

```yaml
# Atributos del sensor:
power_profile_watts:
  - 0      # Hora 1 (próxima hora)
  - 0      # Hora 2
  ...
  - 3600   # Horas de carga
  ...
  - 0      # Hora 168
deferrables_schedule: |  # Lista con timestamps
  [{"date": "2026-04-09T14:00:00+02:00", "p_deferrable0": "3600.0"}, ...]
emhass_status: "ready"  # o "active", "idle", "computing"
```
