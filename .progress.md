## 2026-04-24: Backward Charge Deficit Propagation Decomposition

### Key Decomision Decisions

1. **Spec A is a pure function** — no external dependencies, fully testable in isolation. Lives in `calculations.py` alongside `calculate_multi_trip_charging_windows()` and `calculate_deficit_propagation()`.

2. **Spec B separates wiring from capping fix** — the capping guard in `_populate_per_trip_cache_entry` (lines 638-642) silently discards overflow. The fix is to NOT cap and let EMHASS handle it, while logging a warning.

3. **No separate panel spec** — panel.py reads sensor attributes (`def_total_hours_array`) which update automatically when the cache entries change. The propagation effect cascades through the sensor layer without panel changes.

4. **Propagation walks backward (last trip first)** — matches the existing `calculate_deficit_propagation()` pattern. Each trip's deficit is added to the previous trip's `incoming_deficit`.

### Interface Contracts That Emerged

- `calculate_hours_deficit_propagation()` returns enriched window dicts with 3 new fields: `deficit_hours_propagated`, `deficit_hours_to_propagate`, `adjusted_def_total_hours`
- Input windows must have `ventana_horas`, `horas_carga_necesarias`, `es_suficiente`, and `trip` keys
- The function does NOT modify `ventana_horas` — it only computes scheduling parameters (`def_total_hours`)
- `emhass_adapter.py` wiring adds a call between `calculate_multi_trip_charging_windows()` and the batch loop that populates `batch_charging_windows`

### Risks Identified

1. EMHASS may reject `def_total_hours > window_size` — logged but not prevented
2. Chain propagation depth — N trips could mean trip 0 receives deficit from all N-1 subsequent trips
3. Mocking complexity for integration tests — need to craft trip data that creates desired deficit patterns

### Dependencies Between Specs

- Spec B depends on Spec A (imports `calculate_hours_deficit_propagation`)
- Spec C tests both A and B (unit tests for A, integration tests for B)
- All specs are vertical slices — each delivers user value independently if needed
